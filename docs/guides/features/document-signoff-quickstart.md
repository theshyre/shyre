# Document sign-off — quick guide

Get a document signed off in five steps.

1. **Sign-offs → New sign-off** (or press `N`). Owner/admin only.
2. Add a **title** and **version** (e.g. `Release Notes v2.0.2`), and paste the
   document as **Markdown**.
3. Add your **signatories** (name + email; role and organization optional).
   They can be anyone — your team, the client, a sponsor.
4. **Create draft**, then **Send for signature**. Each signatory is emailed a
   private link.
5. Each signatory verifies with a one-time emailed code, reviews the document,
   types their name, and **signs**. When everyone has signed, the sign-off is
   **Completed** — with an immutable, content-hashed record of each signature.

See the [full guide](document-signoff.md) for signing modes, the signed record,
canceling, and the 21 CFR Part 11 grade.
